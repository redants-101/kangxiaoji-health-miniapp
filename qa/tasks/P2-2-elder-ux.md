## 任务：P2-2 适老化深度优化

- 级别：P2
- 影响范围：`app.wxss`、`pages/home/index.wxss`、`pages/record/record-bp/index.js`、`pages/record/record-bg/index.js`、`components/big-number-input/index.wxml`
- 功能点：
  1. `ui-scale-elder` 模式下所有可点击元素 `min-height` 提升至 `120rpx`，间距增加 50%
  2. 表单验证错误信息在老年模式下使用 `notice` 组件替代行内小字
  3. 时间输入增加"快捷时间"选项（现在/晨起/睡前），减少 picker 操作
  4. 血糖场景标签与时间联动（空腹→06:00，餐后→当前+2h）
  5. 首页待办面板在老年模式下增大行高和字号
- 技术难点：
  1. 适老化样式覆盖面广，需逐页面检查和调整
  2. 快捷时间选项需与现有 picker 共存，不破坏原有交互
  3. 场景标签与时间联动需用户可覆盖（用户选择"空腹"后仍可手动改时间）
- 验收标准：
  1. `ui-scale-elder` 模式下所有按钮 `min-height >= 120rpx`
  2. `ui-scale-elder` 模式下所有行间距 >= `40rpx`
  3. 血压/血糖记录页在老年模式下错误信息使用 notice 组件展示
  4. 时间输入区域显示"现在""晨起""睡前"快捷按钮
  5. 选择"空腹"标签后时间自动设为 06:00，用户仍可手动修改
  6. 首页待办面板在老年模式下行高 >= `130rpx`
- 回归风险：
  - 适老化样式可能影响普通模式的布局
  - 快捷时间按钮可能遮挡原有 picker 入口
  - 场景标签与时间联动可能干扰用户自定义时间
- 关联任务：P1-1 首页空状态引导（新增的悬浮按钮需适配老年模式）

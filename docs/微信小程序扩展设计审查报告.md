# 微信小程序扩展设计审查报告

审查时间：2026-04-26

审查依据：
- 微信小程序设计指南
- 小程序大屏适配指南
- 小程序适老化设计指南
- 小程序无障碍设计指南

审查范围：
- 当前小程序页面、公共样式、公共组件。
- 重点检查大屏适配、适老化、无障碍三类扩展规范。

## 1. 结论概览

当前项目已经补齐了基础状态页和主链路，但从扩展设计规范看，仍然明显偏“手机标准字号原型”，距离适老化、大屏和无障碍达标还有一段距离。

最集中的问题有 5 类：
- 没有接入微信字体档位适配。
- 多处点击热区偏小。
- 多个关键页面仍是固定三列布局。
- 图表页依赖固定坐标摆放视觉元素。
- 自定义界面没有补无障碍语义。

## 2. 主要问题

### 2.1 未接入微信字体档位与适老化缩放

证据：
- 全项目检索 `fontSizeSetting`、`fontSizeScaleFactor`、`getAppBaseInfo`、`page-meta`、`onResize`、`windowWidth` 结果为 0。
- 公共字号大量写死在 [app.wxss](../app.wxss) 中，例如 24rpx、25rpx、26rpx、固定 3 列网格和固定 92rpx 输入框高度。

影响：
- 无法根据微信字体设置做等比放大。
- 关怀模式或大字号模式下，文字、容器和间距不会联动适配。

### 2.2 多处高频操作热区偏小

证据：
- 首页提醒按钮只有 64rpx 高：[pages/home/index.wxss](../pages/home/index.wxss)
- 历史记录筛选项只有 72rpx 高：[pages/record-list/index.wxss](../pages/record-list/index.wxss)
- 提醒中心筛选项只有 72rpx 高：[pages/reminder/index.wxss](../pages/reminder/index.wxss)
- 周报分享按钮只有 64rpx 高：[pages/report/index.wxss](../pages/report/index.wxss)
- 数据管理头部按钮只有 64rpx 高：[pages/data/index.wxss](../pages/data/index.wxss)

判断：
- 这些尺寸对标准成人用户还能用，但对高龄用户和抖动手势用户不友好。
- 结合官方适老化热区建议，当前项目的小胶囊按钮和头部按钮是首批需要放大的区域。

### 2.3 多个关键页面仍使用固定三列布局

证据：
- 首页快速入口固定三列：[pages/home/index.wxss](../pages/home/index.wxss)
- 个人中心统计区固定三列：[pages/me/index.wxss](../pages/me/index.wxss)
- 周报摘要固定三列：[pages/report/index.wxss](../pages/report/index.wxss)
- 数据管理摘要固定三列：[pages/data/index.wxss](../pages/data/index.wxss)
- 历史记录摘要固定三列：[pages/record-list/index.wxss](../pages/record-list/index.wxss)

影响：
- 大屏下只会把空白拉大，不能利用额外宽度。
- 大字号模式下，三列内容容易被压缩到难读。
- 不符合大屏适配指南强调的动态布局与响应策略。

### 2.4 趋势页和周报页图表采用固定坐标摆位

证据：
- 趋势图表点位和折线使用固定 `left/right/top` 数值：[pages/trend/index.wxss](../pages/trend/index.wxss)
- 周报图表同样使用固定 `left/right/top` 数值：[pages/report/index.wxss](../pages/report/index.wxss)
- 对应页面结构是纯视觉 `view` 组合：[pages/trend/index.wxml](../pages/trend/index.wxml)、[pages/report/index.wxml](../pages/report/index.wxml)

影响：
- 视图宽度变化后，点位不会按比例重排。
- 在 PC 端窗口变宽时，图表更像“固定稿件”，不是可伸缩图表。
- 在大字号模式下，也缺少文本替代或布局回退策略。

### 2.5 自定义界面没有补无障碍语义

证据：
- 全项目检索 `aria-` 结果为 0。
- 自定义组件和图表区域较多，例如 [components/quick-action/index.wxml](../components/quick-action/index.wxml)、[components/metric-card/index.wxml](../components/metric-card/index.wxml)、[pages/trend/index.wxml](../pages/trend/index.wxml)、[pages/report/index.wxml](../pages/report/index.wxml)。

影响：
- 非原生界面的语义信息不足。
- 读屏用户只能获取有限文本，难以理解卡片关系、图表含义和状态层级。

## 3. 优先级建议

建议按下面顺序推进：
1. 先补全局能力：读取字体档位、建立缩放系数、统一最小热区。
2. 再改页面布局：把固定三列改成可换行或断点布局。
3. 再重做图表：至少保证宽度变化时点位按比例布局，并提供文本摘要。
4. 最后补无障碍：为自定义组件与关键区域增加语义标注。

## 4. 最值得先动的页面

如果只做第一轮，建议优先处理：
- 首页
- 历史记录
- 提醒中心
- 周报
- 趋势
- 个人中心
- 数据管理

原因：
- 这些页面同时命中了高频访问、小热区、固定三列或固定坐标中的至少两项。

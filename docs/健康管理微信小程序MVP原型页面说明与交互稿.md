# 健康管理微信小程序 MVP 原型页面说明与交互稿

版本：V1.0  
编制日期：2026-04-17  
关联文档：[健康管理微信小程序MVP产品需求文档.md](./健康管理微信小程序MVP产品需求文档.md)

## 1. 原型目标

本文件用于把 PRD 拆解为微信小程序低保真页面原型和交互说明，供设计、开发、测试直接对齐页面结构和交互行为。

MVP 原型坚持三个原则：

- 先完成记录、提醒、家属共享、趋势和周报闭环。
- 所有健康提示只做记录提醒和复测建议，不做诊断、治疗、处方、补服或急救判断。
- 面向中老年用户优先保证大字、少步骤、强反馈和低学习成本。

## 2. 信息架构

### 2.1 底部导航

MVP 使用 4 个 Tab，保证常用入口稳定。

| Tab | 图标建议 | 页面 | 说明 |
| --- | --- | --- | --- |
| 首页 | home | P-HOME | 今日待办、快速记录、最新数据 |
| 趋势 | chart | P-TREND | 7/30/90 天趋势和历史列表 |
| 家庭 | users | P-FAMILY | 家属邀请、授权、家庭成员 |
| 我的 | user | P-ME | 设置、隐私、反馈、帮助 |

说明：

- “记录”不放入 Tab，通过首页主按钮进入，减少底部导航数量。
- “用药”从首页今日待办和快捷入口进入。
- “提醒中心”从首页右上角铃铛和我的页进入。

### 2.2 页面地图

```text
启动
  -> 隐私确认 P-PRIVACY
  -> 隐私详情 P-PRIVACY-DETAIL
  -> 角色选择 P-ROLE
  -> 等待家属邀请 P-FAMILY-JOIN-HINT
  -> 基础资料 P-PROFILE
  -> 首页 P-HOME

首页 P-HOME
  -> 血压录入 P-RECORD-BP
  -> 血糖录入 P-RECORD-BG
  -> 用药计划 P-MED-LIST
  -> 服药确认 P-MED-CONFIRM
  -> 提醒中心 P-REMINDER
  -> 记录详情 P-RECORD-DETAIL

趋势 P-TREND
  -> 记录详情 P-RECORD-DETAIL
  -> 周报 P-REPORT

家庭 P-FAMILY
  -> 邀请家属 P-FAMILY-INVITE
  -> 家属加入确认 P-FAMILY-JOIN
  -> 授权管理 P-FAMILY-AUTH

我的 P-ME
  -> 提醒设置 P-REMINDER-SETTINGS
  -> 隐私与授权 P-PRIVACY-SETTINGS
  -> 数据管理 P-DATA
  -> 帮助中心 P-HELP
  -> 意见反馈 P-FEEDBACK
```

## 3. 全局交互规则

### 3.1 页面结构

标准页面结构：

```text
[导航栏：标题 + 可选右侧图标]
[关键状态区：今日、用户、指标摘要]
[主要内容区：列表、表单或图表]
[固定底部主按钮：保存/确认/邀请/完成]
```

适老化要求：

- 页面主标题 20px 以上。
- 关键数字 32px 以上。
- 主要按钮高度 48px 以上。
- 表单项上下间距不低于 16px。
- 单页主操作不超过 2 个。

### 3.2 通用按钮

| 类型 | 样式建议 | 用途 |
| --- | --- | --- |
| 主按钮 | 微信绿实心，底部固定 | 保存、确认、邀请 |
| 次按钮 | 白底描边 | 跳过、稍后、取消 |
| 危险按钮 | 红色文字，不用大面积红底 | 删除、解绑、注销 |
| 图标按钮 | 铃铛、设置、编辑、删除 | 工具操作 |

### 3.3 通用反馈

| 场景 | 反馈 |
| --- | --- |
| 保存成功 | Toast：已保存 |
| 保存失败 | Toast：保存失败，请稍后再试 |
| 网络异常 | 页面内提示：网络不稳定，请稍后重试 |
| 无数据 | 空态图标 + 1 句引导 + 主按钮 |
| 删除/解绑/注销 | 二次确认弹窗 |

### 3.4 合规固定提示

异常记录、趋势和周报底部固定展示：

```text
本内容仅基于你记录的数据做提醒，不构成诊断或治疗建议；如持续异常或身体不适，请咨询医生或及时就医。
```

用药页面底部固定展示：

```text
用药信息由用户自行记录，请按医生或药师指导用药。
```

家属端固定展示：

```text
数据来自用户自行记录，仅供家庭管理参考。
```

## 4. 关键流程交互

### 4.1 首次进入流程

```text
打开小程序
  -> 隐私确认
  -> 角色选择
  -> 基础资料
  -> 首页
```

交互规则：

- 未同意隐私指引时，“开始使用”按钮置灰。
- 用户可查看完整隐私指引。
- 角色选择后可在我的页修改，但修改时要提示会影响首页展示。

### 4.2 健康记录流程

```text
首页点击“记血压/记血糖”
  -> 大字录入页
  -> 保存
  -> 记录详情页
  -> 返回首页或继续记录
```

交互规则：

- 数字输入时使用自定义大数字键盘或系统数字键盘。
- 保存前做范围校验。
- 明显异常时不阻止保存，但在详情页给复测提示。
- 保存后首页最新记录立即刷新。

### 4.3 用药确认流程

```text
首页今日待办
  -> 点击待确认药品
  -> 服药确认页
  -> 已服 / 跳过 / 稍后
  -> 返回首页
```

交互规则：

- “已服”直接完成。
- “跳过”弹出确认，提示请按医生或药师指导用药，不给补服建议。
- “稍后”默认 15 分钟后再次进入提醒中心，不强制订阅消息。

### 4.4 家属绑定流程

```text
被照护人家庭页
  -> 邀请家属
  -> 选择授权范围
  -> 分享邀请链接
家属打开链接
  -> 查看授权说明
  -> 确认加入
  -> 家属首页
```

交互规则：

- 邀请链接有效期 24 小时。
- 家属加入前必须展示授权范围。
- 被照护人可随时解绑。
- 解绑后家属端看到“授权已取消”空态。

## 5. 页面原型说明

### 5.1 P-PRIVACY 隐私确认页

用户：全部  
优先级：P0  
入口：首次进入小程序

页面目标：完成隐私指引确认，阻止未授权采集健康数据。

低保真：

```text
+--------------------------------+
| 健康记录助手                   |
|                                |
| 记录血压血糖、提醒用药、       |
| 和家人一起查看趋势。           |
|                                |
| [ ] 我已阅读并同意             |
|     《用户隐私保护指引》       |
|                                |
| 说明：本工具用于记录和提醒，    |
| 不提供诊断或治疗建议。         |
|                                |
| [开始使用]                     |
+--------------------------------+
```

交互：

- 勾选同意后，“开始使用”可点击。
- 点击隐私指引打开 P-PRIVACY-DETAIL。
- 点击开始后写入 privacy_accept 日志，进入 P-ROLE。

状态：

- 未勾选：主按钮置灰。
- 写入失败：Toast“请检查网络后重试”。

### 5.2 P-ROLE 角色选择页

用户：全部  
优先级：P0  
入口：隐私确认后

页面目标：区分本人使用和家属使用。

低保真：

```text
+--------------------------------+
| 你想如何使用？                 |
|                                |
| [本人使用]                     |
| 记录自己的血压血糖和用药       |
|                                |
| [帮家人管理]                   |
| 查看家人授权后的记录和提醒     |
|                                |
| [下一步]                       |
+--------------------------------+
```

交互：

- 点击卡片选中角色。
- 本人使用进入 P-PROFILE。
- 帮家人管理进入 P-FAMILY-JOIN-HINT，引导等待邀请或扫描邀请。

状态：

- 未选择角色时下一步置灰。
- 家属未加入家庭组时，首页展示等待授权空态。

### 5.3 P-PRIVACY-DETAIL 隐私详情页

用户：全部  
优先级：P0  
入口：隐私确认页、我的页隐私与授权

页面目标：展示用户隐私保护指引摘要和完整内容入口。

低保真：

```text
+--------------------------------+
| 用户隐私保护指引               |
|                                |
| 我们会处理的信息               |
| - 健康记录                     |
| - 用药提醒                     |
| - 家庭授权关系                 |
| - 订阅消息状态                 |
|                                |
| 使用目的                       |
| 用于记录、提醒、趋势和家庭共享 |
|                                |
| [查看完整指引]                 |
+--------------------------------+
```

交互：

- 从首次隐私确认页进入时，返回后保留勾选状态。
- 从我的页进入时，仅查看，不改变同意状态。

### 5.4 P-FAMILY-JOIN-HINT 等待家属邀请页

用户：家属  
优先级：P0  
入口：角色选择“帮家人管理”

页面目标：提示家属需要被照护人邀请后才能查看数据。

低保真：

```text
+--------------------------------+
| 等待家人邀请                   |
|                                |
| 请让家人在小程序的“家庭”页面   |
| 邀请你加入。加入后，你只能查看 |
| 对方授权给你的内容。           |
|                                |
| [我已收到邀请]                 |
| [返回首页]                     |
+--------------------------------+
```

交互：

- 点击我已收到邀请，提示通过微信聊天中的邀请链接打开。
- 未加入家庭组时，家属首页保持等待授权空态。

### 5.5 P-PROFILE 基础资料页

用户：被照护人  
优先级：P0  
入口：角色选择本人使用

页面目标：收集最小资料，避免过度采集。

低保真：

```text
+--------------------------------+
| 基础资料                       |
|                                |
| 称呼                           |
| [ 例如：妈妈 ]                 |
|                                |
| 出生年份，可选                 |
| [ 1965 ]                       |
|                                |
| 主要关注                       |
| [x] 血压  [x] 血糖  [x] 用药   |
|                                |
| [完成]                         |
+--------------------------------+
```

交互：

- 称呼必填，最多 12 字。
- 出生年份可跳过。
- 至少选择一个关注项。
- 完成后进入 P-HOME。

### 5.6 P-HOME 首页，被照护人视角

用户：被照护人  
优先级：P0  
入口：底部 Tab 首页

页面目标：完成今日待办、快速记录和最新数据查看。

低保真：

```text
+--------------------------------+
| 今天                    [铃铛] |
| 4月17日                         |
|                                |
| 今日待办                       |
| [08:00 服药待确认]       [去确认]|
| [今晚 记一次血压]        [去记录]|
|                                |
| 快速记录                       |
| [记血压] [记血糖] [用药]       |
|                                |
| 最新记录                       |
| 血压  128/82 mmHg      07:30   |
| 血糖  6.1 mmol/L       昨天    |
|                                |
| 本周概览                       |
| 已记录 4 天    已服药 85%      |
|                                |
| Tab: 首页 趋势 家庭 我的       |
+--------------------------------+
```

交互：

- 点击铃铛进入 P-REMINDER。
- 点击记血压进入 P-RECORD-BP。
- 点击记血糖进入 P-RECORD-BG。
- 点击用药进入 P-MED-LIST。
- 点击最新记录进入 P-RECORD-DETAIL。
- 点击待确认服药进入 P-MED-CONFIRM。

空态：

- 无健康记录：展示“还没有记录，今天先记一次”，按钮“记血压”“记血糖”。
- 无用药计划：今日待办不展示服药事项，展示入口“添加用药提醒”。

### 5.7 P-HOME-FAMILY 首页，家属视角

用户：家属  
优先级：P0  
入口：底部 Tab 首页，家属已加入家庭组

页面目标：让家属快速了解被照护人的最近记录和未确认事项。

低保真：

```text
+--------------------------------+
| 家人健康                 [铃铛]|
| 当前查看：妈妈          [切换] |
|                                |
| 今日状态                       |
| [有 1 项用药未确认]            |
|                                |
| 最新记录                       |
| 血压  128/82 mmHg      07:30   |
| 血糖  6.1 mmol/L       昨天    |
|                                |
| 快捷查看                       |
| [趋势] [用药确认] [周报]       |
|                                |
| 数据来自用户自行记录，仅供家庭 |
| 管理参考。                     |
|                                |
| Tab: 首页 趋势 家庭 我的       |
+--------------------------------+
```

交互：

- 点击切换打开被照护人选择底部弹层。
- 家属只能查看，不能编辑健康记录。
- 如果授权被取消，展示“授权已取消，请联系家人重新邀请”。

### 5.8 P-RECORD-BP 血压录入页

用户：被照护人  
优先级：P0  
入口：首页“记血压”

页面目标：30 秒内完成血压记录。

低保真：

```text
+--------------------------------+
| 记血压                         |
|                                |
| 收缩压                         |
| [ 128 ]  mmHg                  |
|                                |
| 舒张压                         |
| [ 82  ]  mmHg                  |
|                                |
| 心率，可选                     |
| [ 72  ]  次/分                 |
|                                |
| 测量时间  今天 07:30    [修改]|
| 场景标签  [晨起] [睡前] [其他]|
| 备注      [可不填]             |
|                                |
| [保存]                         |
+--------------------------------+
```

交互：

- 输入框聚焦时显示数字键盘。
- 收缩压、舒张压必填。
- 心率可为空。
- 点击修改时间弹出时间选择器。
- 点击保存后进入 P-RECORD-DETAIL。

校验：

- 收缩压范围 60-260。
- 舒张压范围 40-160。
- 收缩压必须大于舒张压。
- 心率范围 30-220。

错误提示：

- “收缩压需要大于舒张压，请检查后保存。”
- “这个数值看起来不太常见，请确认是否输入正确。”

### 5.9 P-RECORD-BG 血糖录入页

用户：被照护人  
优先级：P0  
入口：首页“记血糖”

页面目标：快速完成血糖记录，并标记测量场景。

低保真：

```text
+--------------------------------+
| 记血糖                         |
|                                |
| 血糖值                         |
| [ 6.1 ]  mmol/L                |
|                                |
| 测量场景                       |
| [空腹] [餐前] [餐后] [睡前]   |
| [其他]                         |
|                                |
| 测量时间  今天 07:30    [修改]|
| 备注      [可不填]             |
|                                |
| [保存]                         |
+--------------------------------+
```

交互：

- 血糖值必填，支持 1 位小数。
- 测量场景必选。
- 保存后进入 P-RECORD-DETAIL。

校验：

- 血糖范围 1.0-33.3 mmol/L。
- 超出范围提示确认，不直接保存。

### 5.10 P-RECORD-DETAIL 记录详情页

用户：全部  
优先级：P0  
入口：保存后、首页最新记录、趋势点位、记录列表

页面目标：展示单条记录、保守提示和编辑入口。

低保真：

```text
+--------------------------------+
| 记录详情                 [编辑]|
|                                |
| 血压                           |
| 128 / 82 mmHg                  |
| 心率 72 次/分                  |
| 今天 07:30  晨起               |
|                                |
| 提醒                           |
| 本次记录在常见范围内，建议继续 |
| 保持记录。                     |
|                                |
| 近 7 天                        |
| [小趋势图]                     |
|                                |
| 本内容仅基于你记录的数据做提醒 |
| ...                            |
|                                |
| [再记一次]                     |
+--------------------------------+
```

交互：

- 被照护人可编辑、删除。
- 家属端隐藏编辑和删除，只显示查看。
- 点击再记一次进入对应录入页。

异常状态：

- 偏高：展示建议复测和咨询医生。
- 明显偏高：展示“如伴随不适，请及时就医”，不触发电话或急救流程。

### 5.11 P-RECORD-LIST 记录列表页

用户：全部  
优先级：P0  
入口：趋势页、记录详情、首页更多

页面目标：查看历史记录并进入详情。

低保真：

```text
+--------------------------------+
| 历史记录                       |
| [血压] [血糖]                  |
|                                |
| 4月17日                        |
| 07:30  血压 128/82   晨起  >   |
|                                |
| 4月16日                        |
| 21:10  血糖 6.1      睡前  >   |
|                                |
| [加载更多]                     |
+--------------------------------+
```

交互：

- 切换血压/血糖刷新列表。
- 点击记录进入详情。
- 下拉刷新，触底加载更多。

空态：

- “还没有记录，先记一次血压或血糖。”

### 5.12 P-MED-LIST 用药计划页

用户：被照护人  
优先级：P0  
入口：首页“用药”

页面目标：管理用药计划，查看今日服药确认。

低保真：

```text
+--------------------------------+
| 用药提醒                [+]    |
|                                |
| 今日待确认                     |
| 08:00  二甲双胍  1片     [确认]|
| 20:00  降压药    1片     [待到点]|
|                                |
| 用药计划                       |
```

- 点击编辑进入 P-MED-EDIT。

空态：

- “还没有用药提醒”，按钮“添加用药”。

### 5.13 P-MED-EDIT 添加/编辑用药页

用户：被照护人  
优先级：P0  
入口：用药计划页 + 或编辑

页面目标：创建用药计划，不提供药品建议。

低保真：

```text
+--------------------------------+
| 添加用药                       |
|                                |
```

## 6. 组件原型

### 6.1 BigNumberInput 大字数字输入

用途：血压、血糖、心率录入。

结构：

```text
标签
[ 大号数字输入框 ] 单位
辅助提示
```

交互：

- 聚焦后全选当前值。
- 支持清空。
- 错误时边框变红，底部展示错误文字。

### 6.2 MetricCard 指标卡

用途：首页最新记录、家属首页摘要。

结构：

```text
指标名称
大号数值 + 单位
时间/标签
状态标记
```

状态：

- 正常：不显示诊断性文字。
- 偏高/偏低：显示“建议复测”。
- 无数据：显示“暂无记录”。

### 6.3 ReminderItem 提醒项

用途：首页今日待办、提醒中心。

结构：

```text
时间 + 类型
标题
状态
操作按钮
```

状态：

- 待确认。
- 已完成。
- 已过期未确认。
- 已读。

### 6.4 ConsentPanel 授权说明面板

用途：隐私确认、家属加入、授权管理。

必须包含：

- 授权对象。
- 授权范围。
- 数据用途。
- 撤销方式。
- 确认按钮。

## 10. 原型验收清单

### 10.1 产品验收

- 首页首屏能看到记血压、记血糖、用药入口。
- 被照护人和家属首页视角不同。
- 血压和血糖录入路径不超过 3 步。
- 用药跳过不出现补服建议。
- 家属加入前能看到授权范围。
- 家属解绑后能看到授权取消状态。
- 趋势和周报不出现诊断结论。

### 10.2 适老化验收

- 关键数字足够大。
- 主按钮始终清晰可见。
- 输入错误提示使用直接中文。
- 单页不堆叠过多功能。
- 小屏手机不遮挡底部按钮。

### 10.3 合规验收

- 页面中不出现“诊断报告”“治疗方案”“处方建议”“补服方案”等高风险功能入口。
- 异常记录只使用“建议复测”“咨询医生”“及时就医”。
- 隐私确认在采集健康数据前完成。
- 家属授权可查看、可撤销、可留痕。
- 反馈页提示不要提交病历、身份证等敏感内容。

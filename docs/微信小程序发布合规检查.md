# 微信小程序发布合规检查

更新时间：2026-04-27

## 1. 服务定位

康小记定位为“家庭健康记录与提醒工具”，不是医疗机构、互联网医院、问诊平台、处方平台、药品销售平台或急救服务。

建议提交功能描述：

> 康小记用于用户手动记录血压、血糖、用药计划和服药确认，支持提醒、趋势回顾、周报摘要及经授权的家庭成员查看。所有提示仅用于记录和复测提醒，不提供诊断、治疗、处方、在线问诊、购药或急救判断。

## 2. 类目选择建议

微信公众平台后台类目会随平台规则变化，最终以提审时后台可选类目为准。

- 优先选择非诊疗属性的健康管理、健康工具、记录工具或效率工具类目。
- 如果只能选择医疗健康相关类目，应选择“健康管理/健康工具/健康记录”这类非诊疗子类，不选择“互联网医院、在线问诊、处方流转、药品销售、医疗器械、医生服务”等子类。
- 提审截图、功能说明、页面文案要统一强调“记录、提醒、家属共享、趋势回顾”，避免让审核认为存在医疗诊断、处方、问诊或购药能力。

## 4. 数据权利闭环

| 用户权利 | 当前实现 | 云函数 action |
|---|---|---|
| 导出数据 | 数据管理页“复制完整个人数据”导出 JSON 文本 | `exportUserData` |
| 删除健康数据 | 删除健康记录、用药确认和健康统计 | `deleteUserData` |
| 清空账号 | 清理个人资料、记录、用药、家庭关系、提醒、隐私和反馈 | `clearUserAccount` |
| 撤销家属授权 | 家庭页解除授权后，家属首页不再返回授权数据 | `revokeFamilyMember` |

## 5. 禁用或高风险表述

| 高风险表述 | 替代表述 |
|---|---|
| AI 诊断、诊断报告 | 健康记录周报、记录回顾 |
| 判断你患有高血压/糖尿病 | 本次记录偏高/偏低，建议复测并咨询医生 |
| 推荐/调整/停用/补服药物 | 请按医生或药师指导用药 |
| 在线问诊、医生咨询 | 帮助反馈、使用问题反馈 |
| 处方、开药、购药 | 用户自填用药计划和提醒 |
| 急救判断 | 如有明显不适或紧急情况，请及时就医或拨打急救电话 |

## 6. 提审前检查项

- 页面标题、按钮、FAQ、周报、趋势页不出现“诊断、治疗方案、处方、问诊、购药、疗效”等能力承诺。
- 用药相关页面只允许用户自填药品名称、剂量说明和提醒时间，不推荐药品、不建议补服。
- 趋势和周报只展示记录统计、偏高/偏低提醒和复测建议，不给疾病结论。
- 家属端只能查看授权范围内的信息，不能编辑被授权人的健康记录。
- 隐私政策、用户协议和首次隐私确认页的采集范围保持一致。

# 健康管理微信小程序 MVP 产品需求文档

版本：V1.0  
编制日期：2026-04-17  
关联文档：[健康管理微信小程序独立开发可行性分析.md](./健康管理微信小程序独立开发可行性分析.md)

## 1. 产品概述

### 1.1 产品定位

本产品定位为“家庭慢病记录与提醒工具”，服务中国大陆微信生态下的中老年慢病或慢病前期用户，以及负责照看的子女家属。MVP 只解决日常记录、用药提醒、家属查看和趋势回顾问题，不提供诊断、治疗、处方、补服建议、急救判断和药品销售。

一句话描述：让老人更容易记录血压血糖和服药情况，让家属更及时了解长期趋势。

### 1.2 目标用户

| 用户类型 | 用户画像 | 核心需求 | 产品价值 |
| --- | --- | --- | --- |
| 被照护人 | 50 岁以上，高血压、糖尿病或慢病前期人群 | 方便记录、按时服药、看懂趋势 | 大字录入、提醒、趋势图 |
| 家属 | 子女、配偶、照护者 | 远程了解记录和漏服情况 | 家庭绑定、周报、提醒中心 |
| 独立开发者/运营者 | 1-2 人小团队 | 快速上线并验证留存 | 低合规风险、低技术复杂度 |

### 1.3 MVP 目标

MVP 的目标不是验证医疗效果，而是验证家庭健康管理工具的使用意愿和留存。

核心目标：

- 被照护人能在 30 秒内完成一次血压或血糖记录。
- 被照护人能创建用药提醒并确认已服药。
- 家属能在老人授权后查看健康记录、用药确认和周报。
- 系统能展示 7 天、30 天、90 天趋势。
- 系统能对明显异常记录给出保守提示：建议复测并咨询医生。

### 1.4 成功指标

| 指标 | 口径 | MVP 目标 |
| --- | --- | --- |
| 7 日记录率 | 新用户注册后 7 天内至少记录 3 天 | >= 35% |
| 30 日留存 | 第 30 天仍有记录、查看或确认动作 | >= 20% |
| 家属绑定率 | 被照护人绑定至少 1 名家属 | >= 30% |
| 提醒确认率 | 应确认服药提醒中被标记为已服/跳过的比例 | >= 50% |
| 周报打开率 | 周报生成后 7 天内被打开 | >= 40% |

## 2. 范围定义

### 2.1 MVP 包含

| 模块 | 功能 | 优先级 |
| --- | --- | --- |
| 用户与角色 | 微信登录、角色选择、基础资料 | P0 |
| 健康记录 | 血压记录、血糖记录、历史列表、编辑删除 | P0 |
| 用药提醒 | 用药计划、提醒时间、服药确认、跳过记录 | P0 |
| 家属协同 | 邀请家属、授权查看、解绑、家庭首页 | P0 |
| 趋势分析 | 7/30/90 天趋势图、日历记录状态 | P0 |
| 异常提示 | 基于固定阈值的复测提示和就医提示 | P0 |
| 周报 | 每周记录次数、趋势变化、漏服次数、家属查看摘要 | P1 |
| 提醒中心 | 测量提醒、服药提醒、周报通知、未确认提醒 | P1 |
| 隐私合规 | 隐私指引、家属授权、数据删除、注销入口 | P0 |
| 基础运营 | 帮助中心、意见反馈、免责声明 | P1 |

### 2.2 MVP 不包含

- 不做在线问诊。
- 不做医生诊断。
- 不做处方流转。
- 不做药品销售。
- 不做药物相互作用自动判断。
- 不做漏服后的补服建议。
- 不做自动调整药量。
- 不做红色紧急预警和电话外呼。
- 不做蓝牙设备接入。
- 不做 OCR 读数识别。
- 不做 AI 健康报告。
- 不做社区/药店机构后台。

### 2.3 合规文案原则

| 禁用表达 | MVP 替代表达 |
| --- | --- |
| AI 诊断报告 | 健康记录周报 |
| 系统判断您患有高血压 | 本次记录数值偏高，建议复测并咨询医生 |
| 建议立即服用某药 | 请按医生或药师指导用药 |
| 漏服后请补服 | 发现本次未确认服药，请查看医嘱或咨询医生 |
| 治疗方案 | 生活方式科普 |
| 降压效果显著 | 帮助记录和回顾血压变化 |

所有异常提示统一附加：本提示仅基于你记录的数据做提醒，不构成诊断或治疗建议；如持续异常或身体不适，请咨询医生或及时就医。

## 3. 用户流程

### 3.1 首次进入流程

1. 用户进入小程序。
2. 展示产品说明和免责声明，说明产品用于记录和提醒，不用于诊断治疗。
3. 用户授权微信登录。
4. 用户选择角色：本人使用 / 帮家人管理。
5. 用户填写最小资料：称呼、出生年份可选、主要关注项血压/血糖/用药。
6. 用户进入首页。

验收标准：

- 不强制收集手机号、身份证、病史、医保信息。
- 用户未同意隐私指引时，不进入健康数据录入流程。
- 首次进入路径不超过 4 个页面。

### 3.2 健康记录流程

1. 用户在首页点击“记血压”或“记血糖”。
2. 进入大字录入页。
3. 血压记录输入收缩压、舒张压、心率可选。
4. 血糖记录输入血糖值、测量场景：空腹/餐前/餐后/睡前/其他。
5. 用户可选择测量时间，默认当前时间。
6. 用户可添加备注。
7. 保存后进入结果页，显示本次记录、历史对比、保守提示。

验收标准：

- 血压必须包含收缩压和舒张压。
- 血糖必须包含数值和单位 mmol/L。
- 数值超出合理录入范围时提示重新确认。
- 异常提示只建议复测、记录、咨询医生。
- 保存后首页即时更新最新记录。

### 3.3 用药提醒流程

1. 用户点击“添加用药”。
2. 输入药品名称，剂量文本可选。
3. 设置提醒频次：每日一次、每日多次、每周指定日。
4. 设置提醒时间。
5. 选择是否开启订阅消息。
6. 到点后展示订阅消息或小程序内提醒。
7. 用户进入提醒详情，选择“已服”“跳过”“稍后”。

验收标准：

- 剂量只作为用户自填文本保存，不生成建议。
- “跳过”后提示“请按医生或药师指导用药”，不提示补服。
- 订阅消息授权失败时，仍在提醒中心展示未确认事项。
- 过期提醒保留在当日未确认列表中。

### 3.4 家属绑定流程

1. 被照护人在家庭页点击“邀请家属”。
2. 系统生成一次性邀请链接或二维码。
3. 家属打开后看到授权说明。
4. 被照护人确认授权范围：查看健康记录、查看用药确认、接收周报。
5. 家属加入家庭组。
6. 被照护人可随时解绑家属。

验收标准：

- 家属不能绕过被照护人授权查看数据。
- 被照护人可查看所有家属列表和授权范围。
- 解绑后家属不能继续查看新数据。
- 家属端显示“数据来自用户自行记录，仅供家庭管理参考”。

### 3.5 趋势与周报流程

1. 用户进入趋势页。
2. 选择血压、血糖、服药记录。
3. 切换 7 天、30 天、90 天。
4. 查看折线图、列表和异常记录标记。
5. 每周一生成上一周周报。
6. 用户和授权家属可查看周报。

验收标准：

- 趋势页不显示诊断结论。
- 周报只总结记录次数、趋势变化、漏服确认情况。
- 周报固定附加免责声明。

## 4. 功能需求

### 4.1 首页

需求编号：PRD-HOME-001  
优先级：P0

首页展示：

- 今日待办：待测量、待服药、未确认提醒。
- 最新记录：最近一次血压、最近一次血糖。
- 快捷入口：记血压、记血糖、用药、趋势、家庭。
- 家属入口：家属身份时展示被照护人列表和异常/未确认摘要。

验收标准：

- 首页首屏必须能完成血压、血糖、用药三个核心入口。
- 被照护人和家属看到的首页内容不同。
- 无记录时展示空状态和引导，不展示虚假数据。

### 4.2 血压记录

需求编号：PRD-RECORD-BP-001  
优先级：P0

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 收缩压 | 是 | mmHg，整数 |
| 舒张压 | 是 | mmHg，整数 |
| 心率 | 否 | bpm，整数 |
| 测量时间 | 是 | 默认当前时间 |
| 场景标签 | 否 | 晨起、午间、睡前、运动后、其他 |
| 备注 | 否 | 100 字以内 |

录入校验：

- 收缩压合理范围：60-260。
- 舒张压合理范围：40-160。
- 心率合理范围：30-220。
- 收缩压必须大于舒张压。

异常提示规则：

| 条件 | 提示 |
| --- | --- |
| 收缩压 >= 140 或舒张压 >= 90 | 本次记录偏高，建议安静休息后复测；如持续异常，请咨询医生。 |
| 收缩压 >= 180 或舒张压 >= 120 | 本次记录明显偏高，建议尽快复测；如伴随不适，请及时就医。 |
| 收缩压 < 90 或舒张压 < 60 | 本次记录偏低，建议结合身体感受复测；如头晕乏力，请咨询医生。 |

### 4.3 血糖记录

需求编号：PRD-RECORD-BG-001  
优先级：P0

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 血糖值 | 是 | mmol/L，支持 1 位小数 |
| 测量场景 | 是 | 空腹、餐前、餐后、睡前、其他 |
| 测量时间 | 是 | 默认当前时间 |
| 备注 | 否 | 100 字以内 |

录入校验：

- 血糖合理范围：1.0-33.3 mmol/L。
- 输入超过范围时要求用户确认或修改。

异常提示规则：

| 条件 | 提示 |
| --- | --- |
| 血糖 < 3.9 | 本次记录偏低，建议结合身体感受复测；如有不适，请及时咨询医生。 |
| 空腹血糖 >= 7.0 | 本次空腹记录偏高，建议持续记录并咨询医生。 |
| 餐后血糖 >= 11.1 | 本次餐后记录偏高，建议持续记录并咨询医生。 |

### 4.4 用药计划与确认

需求编号：PRD-MED-001  
优先级：P0

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 药品名称 | 是 | 用户自填 |
| 剂量说明 | 否 | 用户自填，如“1 片” |
| 服药时间 | 是 | 支持每日多个时间 |
| 开始日期 | 是 | 默认当天 |
| 结束日期 | 否 | 为空表示长期 |
| 备注 | 否 | 100 字以内 |

状态：

- 待确认。
- 已服。
- 跳过。
- 稍后提醒。
- 已过期未确认。

验收标准：

- 系统不提供剂量推荐。
- 系统不提供补服建议。
- 删除用药计划时保留历史服药确认记录。
- 家属可查看确认状态，但不能替被照护人修改历史记录，除非被照护人授予“协助记录”权限；MVP 默认不开放协助记录。

### 4.5 家庭组

需求编号：PRD-FAMILY-001  
优先级：P0

角色：

| 角色 | 权限 |
| --- | --- |
| 被照护人 | 创建家庭组、邀请家属、解绑家属、管理授权、查看全部数据 |
| 家属 | 查看授权范围内的数据、查看提醒和周报 |

授权范围：

- 查看血压/血糖记录。
- 查看用药确认。
- 查看周报。
- 接收未确认提醒。

验收标准：

- 每个被照护人至少可绑定 3 名家属。
- 邀请链接有效期 24 小时。
- 被照护人解绑家属后，家属端立即失去访问权限。
- 授权和解绑写入授权日志。

### 4.6 趋势页

需求编号：PRD-TREND-001  
优先级：P0

内容：

- 指标切换：血压、血糖、服药。
- 时间维度：7 天、30 天、90 天。
- 图表：折线图或点线图。
- 列表：按时间倒序展示记录。
- 异常标记：仅标记“偏高/偏低/建议复测”，不标记疾病。

验收标准：

- 数据少于 2 条时展示列表和引导，不强行生成趋势结论。
- 图表点位可点击查看原始记录。
- 家属端展示同样趋势，但不能编辑。

### 4.7 周报

需求编号：PRD-REPORT-001  
优先级：P1

周报周期：每周一生成上周一至周日数据。

内容：

- 本周记录天数。
- 血压/血糖记录次数。
- 偏高/偏低记录次数。
- 用药应确认次数、已确认次数、跳过次数、未确认次数。
- 家属查看次数。
- 保守总结文案。

验收标准：

- 周报标题不得使用“诊断”“治疗”“疗效”等词。
- 周报可分享给已授权家属。
- 无数据时生成“本周记录较少”的引导，不生成趋势判断。

### 4.8 提醒中心

需求编号：PRD-REMINDER-001  
优先级：P1

提醒类型：

- 测量提醒。
- 用药提醒。
- 周报提醒。
- 未确认提醒。

验收标准：

- 订阅消息授权失败不影响提醒中心记录生成。
- 提醒按今天、近 7 天、已完成分组。
- 用户可关闭某类提醒。

### 4.9 隐私与账号

需求编号：PRD-PRIVACY-001  
优先级：P0

功能：

- 隐私指引确认。
- 家属授权说明。
- 授权范围查看和撤销。
- 数据删除。
- 账号注销。
- 免责声明展示。

验收标准：

- 未同意隐私指引前，不采集健康数据。
- 删除健康记录需要二次确认。
- 注销账号需要清理个人资料、健康记录、用药计划、家庭关系和授权日志，或按法律要求做不可恢复匿名化。

## 6. 非功能需求

### 6.1 适老化体验

- 关键数字字号不小于 32px。
- 主要按钮高度不小于 48px。
- 录入页不使用复杂弹窗。
- 每页主操作不超过 2 个。
- 不使用低对比度灰字承载关键信息。
- 错误提示使用明确中文，不展示技术错误码。

### 6.2 性能

- 首页首屏加载目标小于 2 秒。
- 健康记录保存后 1 秒内反馈结果。
- 趋势页 90 天数据加载目标小于 2 秒。
- 周报可异步生成，不阻塞页面。

### 6.3 安全与隐私

- 健康数据接口必须校验用户身份和家庭授权。
- 家属只能查看授权用户的数据。
- 服务端日志不得输出完整健康数值与 openid 组合。
- 隐私政策必须覆盖健康记录、家属共享、订阅消息、云服务。
- 数据删除和账号注销必须可用。

## 8. 发布验收

MVP 发布必须满足：

- P0 页面和功能全部完成。

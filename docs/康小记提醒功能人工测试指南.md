# 康小记提醒功能人工测试指南

文档版本：v1.0
测试范围：提醒功能 P0 核心修复 + 提醒中心 & 提醒设置页面 P0-P2 优化
测试环境：微信开发者工具 + 真机（iOS / Android）
测试日期：2026-06-22

## 测试前准备

### 环境准备

1. **微信开发者工具**
   - 版本：最新稳定版
   - 登录账号：开发者账号
   - AppID：已绑定小程序

2. **真机测试**
   - iOS：微信最新版本
   - Android：微信最新版本
   - 登录账号：体验版权限账号

3. **云开发环境**
   - 云环境 ID 已配置
   - 云函数已部署（sendDueReminders、healthApi）
   - 数据库集合已创建（medication_plans、medication_confirmations、reminder_settings、subscribe_push_logs）

### 测试数据准备

1. **添加测试用药计划**
   - 进入：用药管理 → 添加用药
   - 计划 1：降压药，07:00 和 19:00 各一次，1 片
   - 计划 2：降糖药，08:00 一次，0.5 片
   - 计划 3：维生素，21:00 一次，1 片

2. **清空本地存储**（每次测试前执行）
   - 开发者工具 → Storage → 清除全部
   - 真机：删除小程序后重新进入

3. **重置订阅授权状态**
   - 开发者工具 → 清除全部缓存
   - 真机：小程序设置 → 清除数据

### 测试账号要求

- 测试账号需有"体验版"权限
- 测试账号需在微信公众平台 → 成员管理中添加
- 真机测试时使用体验版二维码扫码

### C. 常见问题排查

**问题 1：授权弹窗不弹出**
- 检查 utils/subscribe-config.js 中 DEV_MODE 是否为 false
- 检查模板 ID 是否正确配置
- 检查是否在 24h 内已引导过（防打扰策略）

**问题 2：snooze 弹窗不弹出**
- 检查 snooze 任务是否到期（dueAt 是否已过）
- 检查是否在 30 秒内已弹过（防打扰策略）
- 检查是否在同一会话内已弹过（_shownSnoozeIds）

**问题 3：云函数推送失败**
- 检查用户是否已授权对应模板
- 检查订阅次数是否充足
- 检查是否已推送过（防重复）
- 查看云函数日志定位错误

**问题 4：任务排序错误**
- 检查任务 time 字段格式（应为 HH:mm）
- 检查 sortTasksByTime 方法是否正确执行
- 检查 activeTab 是否正确

**问题 5：设置保存后未刷新**
- 检查 saveSettings 是否调用了 markDirty
- 检查提醒中心 onShow 是否调用了 loadData
- 检查 _loaded 标志是否正确

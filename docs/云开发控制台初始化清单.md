# 云开发控制台初始化清单

本文用于在微信开发者工具的云开发控制台中手工创建集合和索引。当前环境没有 CloudBase CLI 登录态，控制台操作需要在你的微信开发者工具中完成。

## 1. 新增集合

进入路径：

```text
微信开发者工具 -> 云开发 -> 数据库 -> 集合 -> 新建集合
```

新增集合：

```text
family_members
health_daily_stats
health_record_stats
```

用途：

| 集合 | 用途 |
|---|---|
| `family_members` | 保存真实家庭成员关系，即“数据拥有者 openid”和“家属 openid”的绑定关系。 |
| `health_daily_stats` | 保存健康记录按日预聚合统计，支撑首页今日统计和报告统计。 |
| `health_record_stats` | 保存健康记录按用户总量预聚合统计，支撑首页血压/血糖累计次数。 |

建议权限：

```text
仅管理端可读写
```

原因：`family_members` 涉及跨账号授权关系，不应允许小程序端直接读写，统一通过 `healthApi` 云函数校验。

`health_daily_stats` 和 `health_record_stats` 也属于健康相关个人统计信息，建议同样只允许云函数读写。

## 2. 建议索引

进入路径：

```text
数据库 -> 对应集合 -> 索引管理 -> 新建索引
```

### 2.1 家属账号查可查看家庭

索引名：

```text
idx_member_status_updatedAt
```

字段：

| 字段 | 排序 |
|---|---|
| `memberOpenId` | 升序 |
| `status` | 升序 |
| `updatedAt` | 降序 |

用途：

```js
db.collection('family_members')
  .where({ memberOpenId, status: 'active' })
  .orderBy('updatedAt', 'desc')
```

### 2.2 数据拥有者查看已绑定家属

索引名：

```text
idx_owner_status_updatedAt
```

字段：

| 字段 | 排序 |
|---|---|
| `ownerOpenId` | 升序 |
| `status` | 升序 |
| `updatedAt` | 降序 |

用途：

```js
db.collection('family_members')
  .where({ ownerOpenId, status: 'active' })
  .orderBy('updatedAt', 'desc')
```

### 2.3 防止重复绑定的查询索引

索引名：

```text
idx_owner_member
```

字段：

| 字段 | 排序 |
|---|---|
| `ownerOpenId` | 升序 |
| `memberOpenId` | 升序 |

用途：

```js
db.collection('family_members')
  .where({ ownerOpenId, memberOpenId })
```

### 2.4 健康记录按日统计

集合：`health_daily_stats`

索引名：

```text
idx_openid_dayKey
```

字段：

| 字段 | 排序 |
|---|---|
| `_openid` | 升序 |
| `dayKey` | 降序 |

用途：

```js
db.collection('health_daily_stats')
  .where({ _openid: openId, dayKey: _.gte(startKey) })
  .orderBy('dayKey', 'desc')
```

### 2.5 健康记录用户总统计

集合：`health_record_stats`

索引名：

```text
idx_openid
```

字段：

| 字段 | 排序 |
|---|---|
| `_openid` | 升序 |

用途：按用户读取累计血压/血糖记录数，避免对 `health_records` 做实时 `count()`。

## 3. 验证方式

完成集合和索引后，部署 `cloudfunctions/healthApi`，然后按以下流程验证：

1. 账号 A 进入“家庭 -> 邀请家属”，生成邀请码。
2. 账号 B 打开加入家庭页面，使用邀请码加入。
3. 云数据库 `family_members` 应新增或更新一条记录。
4. 账号 B 进入家属首页，应能看到账号 A 授权范围内的数据。
5. 如线上已有历史健康记录，可调用 `healthApi` 的 `rebuildRecordStats` action 重建当前用户预聚合统计。

期望 `family_members` 记录核心字段：

```js
{
  ownerOpenId: '账号A的openid',
  memberOpenId: '账号B的openid',
  ownerName: '账号A称呼',
  memberName: '女儿',
  inviteCode: 'KXJ...',
  scopes: [],
  noticeRules: [],
  status: 'active',
  joinedAt: Date,
  createdAt: Date,
  updatedAt: Date
}
```
